Chapter 5 Overview of statistical analysis methods

5.1 Multiple linear regression

Linear models are used to compute omics feature-metadata associations for the bulk RNAseq, Nanostring, pseudobulk RNAseq, and bulk proteomics datasets. A linear model containing the selected metadata variable of interest and all covariates (if any) is fit to the expression or abundance levels of each feature in the selected omics dataset, using the limma R package (version 3.52.4). If the user selects a categorical metadata of interest, two dropdowns automatically appear so that a specific contrast can be specified (ie. ‘type 2 diabetes’ versus ‘no diabetes’). Then, the coefficient of the metadata variable (continuous) or metadata contrast (categorical) of interest and its associated p-value are extracted from the model for each feature. P-values are adjusted using the false discovery rate method. The rationale for using the original limma method is that it has been shown to perform well for many omics types as long as the data are appropriately normalized and transformed. While more customized methods developed for specific omics types may perform slightly better when applied to the appropriate data type, the advantage of our approach is that it can be applied to many omics types without modification, making the results more comparable to each other and making the HumanIslets tool easier to maintain. A detailed explanation of how analyze omics data with multiple linear regression, including how to specify linear models for different research questions and how to interpret the results, is available in Protocol #2 here.

5.2 Spearman ranked correlation

Due to the highly nonlinear nature of the relationships between single-cell gene expression and single-cell electrophysiology outcomes (patch-seq), Spearman correlation (base R) is used to compute feature-metadata relationships for the patchSeq data.

5.3 Kendall rank correlation (environmental contaminants)

Environmental contaminant abundances are typically zero-inflated and non-normal, which violates the distributional assumptions used by moderated-t (limma) for the other omics types. For contaminants, the platform therefore substitutes Kendall’s rank correlation. When no covariates are selected, contaminant–phenotype associations are computed with base R’s cor.test(..., method = "kendall"); when covariates are supplied, the same test is run as a partial correlation using ppcor::pcor.test(..., method = "kendall"). Both continuous and dichotomized (0/1) contrasts of the analysis variable are supported. The reported coefficient is Kendall’s tau (not a log-fold-change), and the same Benjamini–Hochberg FDR adjustment is applied to the resulting p-values. Only single-cell electrophysiology outcomes are available for analysis with the single-cell expression data, and covariate adjustment is not supported. For these analyses, electrophysiology parameters other than the half-inactivation sodium current were normalized to cell size. The sign of the calcium entry, sodium current, and calcium current (early and late) electrophysiology parameters was flipped such that greater values correspond to greater currents. Negative exocytosis values were replaced with zeros.

5.4 Pathway analysis

Overrepresentation analysis (ORA) and gene set enrichment analysis (GSEA) are supported for pathway analysis of the feature association results. Both methods are implemented using the fgsea R package (version 1.22.0). The feature list used for ORA is determined by the p-value threshold specified in the association analysis. GSEA can be performed using either the model coefficient (or log2FC when the metadata is categorical) or the test statistic from the association analysis to rank the features. For the patch-seq data, the coefficient is the correlation estimate and there is no test statistic option as it is not compatible with the Spearman correlation results. There are six supported pathway libraries, including KEGG, Reactome, the full Gene Ontology term list (broken into Biological Process, Cellular Component, and Molecular Function), and the PANTHER summary of the Gene Ontology terms (also broken into the same three categories).

Pathway analysis can sometimes return a high number of significant pathways or gene sets that consist of mostly the same features, leading to a redundant results list. Users have the option of collapsing redundant sets, which uses the “collapsePathways” function within the fgsea R package to filter the gene sets to only include those with independent gene lists. Pathway analysis is not supported for the Nanostring data, as there are too few features (n = 156) for adequate pathway coverage.

5.5 Pre-computed feature associations

Two complementary families of associations are pre-computed across the platform and surfaced in the Feature View: univariate (one feature at a time) and multivariate (one omics matrix against a target, jointly).

5.5.1 Univariate associations

Univariate associations were computed between all omics feature-metadata variable pairs and between all metadata variable-metadata variable pairs. The linear model and Spearman correlation methods described above were used to compute the omics-metadata associations, with one modification. Some categorical variables have obvious specific contrasts, for example for diabetes status, one is usually interested in type 1 versus no diabetes and type 2 versus no diabetes. In these cases, we specify each contrast of interest using the methods described above. Other variables do not have obvious comparisons, for example donation type, so in this case an ANOVA style analysis is done and the statistics are extracted for the categorical variable coefficient rather than for a specific contrast. Sex, age, BMI, and culture time were included as covariates for analyses that used a linear model (all bulk omics data associations).

Three different methods were used to compute associations between pairs of metadata, depending on whether each metadata in the pair were continuous or categorical. Pearson correlation was used for continuous-continuous pairs, Cramer’s V correlation was used for categorical-categorical pairs, and ANOVA was used for categorical-continuous pairs where the categorical variable is the independent variable and the continuous variable is the dependent variable. Multiple hypothesis correction (FDR method) was performed across all results for the same statistical method. Some metadata variables such as percent purity and percent trapped are in between ordinal variables and a true continuous variable because they are estimates made by eye by a research technician, and the technician tends to estimate whole percentages (ie. 80%, 70%). For simplicity, we treat these as continuous variables.

5.5.2 Multivariate predictions

In addition to the univariate results, the platform pre-computes multivariate prediction models for each (omics matrix × target phenotype) combination. Where univariate analysis tests each feature in isolation, the multivariate models fit all features in an omics jointly to a target, capturing the predictive signal that distributes across many features. Six complementary methods are run per omics × target so that linear and non-linear signal are both surfaced:

Method Use case Notes
Elastic Net (cross-validated) Continuous targets Linear, sparse feature selection
Logistic Elastic Net (cross-validated) Binary pairwise contrasts Linear classifier with sparse selection
Multinomial Elastic Net (cross-validated) Discrete targets with ≥ 3 levels Linear, multi-class
Stability Selection Any Lasso / Logistic-Lasso refit over 100 random subsamples; selection frequency indicates feature robustness
Random Forest + permutation importance Any Non-linear; importance magnitude only (no signed direction)
XGBoost + SHAP Any Non-linear with per-feature signed direction from SHAP values; GPU-accelerated when available

All six methods run inside a per-(omics, target, method) try/except, so a single failure does not stop the wider pipeline. Per-omics results are written incrementally and resumable, then consolidated into the analysis database for use by the Feature View’s multivariate panel.

5.6 Deconvolution analysis

After normalization for tissue sample size, the intensity of a marker protein that is only abundant in one cell type and that has relatively constant levels is proportional to the abundance of that cell type. We assume an average of 0.55 beta cells, 0.30 alpha cells, 0.10 delta cells, and 0.05 gamma cells and use this to estimate the cell type composition of each donor using proteomics data. We computed the proportion of each cell type for each donor, according to marker genes. The overall proportion of each cell type was calculated as the median of proportion estimates across all marker genes for that cell type. Finally, the proportion of non-endocrine cells was computed as the remainder after adding the estimated proportion of beta, alpha, delta, and gamma cells and normalizing the endocrine proportions to satisfy an assumed maximum purity of 98%. When computing associations between individual omics features and cell type proportions, endocrine cell proportions were normalized as a proportion of the total endocrine signal and the non-endocrine proportion was included as a covariate to better distinguish cell type-specific signals.

Overall, this can be thought of as estimating the strength of the signal associated with four endocrine cell types, anchoring the median signal strength with prior known cell type proportions, and attributing any missing signal to trapped non-endocrine cells. It relies on the assumptions that marker genes are only expressed in one cell type and that their levels are constant across donors. Neither of these assumptions are completely satisfied, however the method should be a robust estimator so long as the majority of marker genes have heavily biased expression in one cell type and if all marker genes are not regulated in the same direction (ie. all beta cell marker genes up-regulated in donors with diabetes).

5.7 Multi-omics integration methods

Three complementary multi-omics methods are precomputed across all eligible omics combinations. Each requires a minimum of 20 shared donors across the omics layers being integrated, and applies per-omics feature caps to keep the inputs comparably weighted (large omics matrices are first ranked by median absolute deviation and capped before integration). See chapter 2 for how to use these methods in the platform; the parameter choices below are extracted directly from the precomputation R scripts.

5.7.1 WGCNA (Weighted Gene Co-expression Network Analysis)

For the combined-omics pipeline, per-omics matrices are first z-scored within each omics, then concatenated into a single feature matrix with prefixed feature IDs (e.g. rnaseq::ENSG…, prot::P00…). A single WGCNA run is then performed on the concatenated matrix, so modules can naturally contain features from multiple omics whenever they co-vary. Per-omics feature caps (MAD-ranked) are: RNA-seq 2,500; proteomics 2,500; pseudobulk 2,500; peak (raw metabolite) 2,000 with a >50% zero filter; metabolite annotated features all kept (~215); flux 1,500 (capped to prevent dense networks dominated by flux). Bicor (biweight midcorrelation, maxPOutliers = 0.1) is used as the correlation method. Networks are built as signed hybrid with TOMType = "signed"; soft threshold is selected by pickSoftThreshold to reach an R² > 0.80 fit to scale-free topology, with a minimum of power = 6 enforced. Module detection adapts to sample size: minModuleSize defaults to 30 and is reduced toward 8 for smaller cohorts; mergeCutHeight defaults to 0.25 (raised to 0.30 for n < 50 donors); deepSplit defaults to 2 (reduced to 1 for n < 50). Module eigengenes are computed by WGCNA::moduleEigengenes(). Hub features are reported with both signed module membership (kME) and intramodular connectivity. Cross-omics structure is reported as the per-module-per-omics variance-reach metric (the mean R² of each module’s eigengene against each omics’ features) and as a flag on TOM edges connecting features of different omics types. Per-module pathway enrichment is performed via fgsea::fora() (ORA) or fgsea::fgsea() (GSEA, ranked by kME); supported libraries include KEGG, the GEM metabolite / reaction libraries, and any GO/Reactome libraries configured.

5.7.2 MOFA (Multi-Omics Factor Analysis)

MOFA is computed with the MOFA2 R package. Unlike WGCNA, donors are taken as the union across omics, with missing-view values represented as NA so the model can learn from partial-overlap donors. Per-omics feature caps (MAD-ranked) are: RNA-seq 5,000; proteomics 5,000; pseudobulk 5,000; flux all reactions; metabolite all annotated; peak 3,000. For metabolite and peak data, ComBat batch correction (two batches B1/B2) is applied before scaling. Features are z-scored row-wise (NA-tolerant) after capping. Up to 15 factors are requested, automatically reduced to min(15, shared_donors / 5, min_features_per_view). All views use a Gaussian likelihood. Training options: scale_views = FALSE (data are already scaled), scale_groups = FALSE, seed = 42; convergence mode is "slow" for fewer than 50 shared donors and "medium" otherwise. Precomputed outputs include per-factor donor scores, per-omics loading matrices, variance explained per factor per view, and the top 20 features per factor per omics by absolute loading.

5.7.3 DIABLO (Data Integration Analysis for Biomarker discovery using Latent cOmponents)

DIABLO is a supervised integration that requires a primary metadata variable as the Y response. It is implemented with the mixOmics R package. For a binary categorical Y the platform uses block.splsda (≥ 3 samples per group required); for a continuous Y it uses block.pls (regression mode). Pre-filtering requires at least 10 donors with measurements in every omics block. Per-omics feature caps (MAD-ranked) are: RNA-seq / pseudobulk 2,000; proteomics 1,000; metabolite 500; contaminants 200; features are z-scored. The number of latent components is min(3, n_features_per_view − 1) for discriminant mode (up to 5 for regression). Blocks are connected by a fixed design matrix with off-diagonal weight 0.2 and diagonal 0 (per-block keepX tuning by cross-validation is implemented but currently disabled in the production pipeline). Outputs precomputed per analysis include: total variance explained per component per omics, a discrimination performance plot (AUROC for binary outcomes; R² for continuous), the top features ranked by max absolute loading across components, and a sample plot in integrated component space (plotArrow).

5.8 Variational Autoencoder (VAE) for donor clustering

The Cluster Explorer uses a custom multi-view Variational Autoencoder pipeline. The pipeline integrates bulk RNA-seq and proteomics for 387 donors (RNA-seq 371, proteomics 134, both 118), the two omics layers with the highest cohort coverage; sparser layers (patch-seq, metabolomics) were intentionally excluded to avoid selection bias. Each omics matrix contributes its 5,000 most-variable features (by cross-donor variance) plus a curated panel of 50 force-included islet biology genes (insulin, PDX1, MAFA, NKX6-1, TCF7L2, HLA loci, stress markers, exocytosis machinery). Before training, every feature is residualized against ten isolation/technical covariates (culture time, purity, % trapped, cold ischemia time, digestion time, collagenase brand/type, donation type, …) by protected linear regression that preserves disease-correlated variance; donor phenotype variables (108 features) are k-nearest-neighbour imputed (k = 10) and z-scored.

The VAE has independent encoder/decoder networks per omics view. Each encoder is two fully-connected layers (512 → 256 units) with batch normalization, ReLU activations, and 20% dropout, projecting onto a 32-dimensional Gaussian latent; decoders mirror the architecture. Multi-modal donor posteriors are fused via a Product-of-Experts rule (combining the precisions of each view’s posterior with a standard-normal prior); single-modality donors are encoded through one expert plus the prior. The objective is per-view mean-squared-error reconstruction plus a Kullback–Leibler divergence term against the standard-normal prior, with linear KL annealing (β: 0 → 1 over 100 epochs) to prevent posterior collapse. Training uses Adam (learning rate 10⁻³, weight decay 10⁻⁴), batch size 64, up to 500 epochs with early stopping (patience 50); the production checkpoint converged at epoch 217 (loss 0.999). Random seed is 42.

Clusters are assigned by k-means (n_init = 20) on the 32-dimensional latent embedding. K-means, Gaussian mixtures, spectral clustering, HDBSCAN, and consensus clustering were benchmarked across k ∈ {3,…,8} using silhouette, Calinski–Harabasz, Davies–Bouldin, and 100-bootstrap (80% donors) adjusted Rand index. K = 5 is the production choice, selected because, although k = 4 had a marginally higher silhouette (0.433 vs 0.413), k = 5 retained a biologically coherent subpopulation (a 29-donor macrophage-signature cluster) and gave balanced cluster sizes (72 / 64 / 167 / 55 / 29). Stability metrics for the final assignment: mean pairwise adjusted Rand index of 0.989 across 20 reseeded runs, 0.971 over bootstrap resamples, and 100% leave-one-out cross-validation accuracy.

To extend cluster labels to the 238 donors without omics data, a separate LightGBM classifier (300 estimators, max_depth = 4, learning_rate = 0.05, subsample = 0.8, colsample_bytree = 0.8) is trained on a deliberately unbiased feature set: demographics (age, BMI, sex), isolation parameters (IEQ, purity, cold ischemia, culture time, digestion time, collagenase brand/type) and pancreas characteristics — 46 input features after one-hot encoding. Missing values are k-NN imputed (k = 10) and z-scored. The classifier is selected from a benchmark against random forest, gradient boosting, and XGBoost by 5-fold stratified cross-validation (5-fold CV accuracy 0.341 ± 0.059, macro-F1 0.226 ± 0.066). Predictions are exposed with native softmax confidence; the platform flags predictions with probability ≥ 0.5 as “high confidence”, which covers 192 of 238 (80.7%) extended donors.

Per-cluster downstream analyses (precomputed) include limma differential expression (one-cluster-vs-rest; RNA-seq FDR < 0.05, proteomics raw p < 0.05 because of small per-cluster sample sizes for proteomics) and pre-ranked GSEA via fgsea v1.30.0 on KEGG_2021_Human, Reactome_2022, and GO_Molecular_Function_2023 gene sets (minSize 10, maxSize 300, FDR < 0.05).

5.9 GEM Library for Pathway Enrichment

Pathway analysis in HumanIslets supports three GEM-based libraries in addition to the standard KEGG / Reactome / Gene Ontology / PANTHER options, selectable from the same library dropdown used for the standard pathway libraries:

  • KEGG human metabolite-centred pathways — KEGG pathways organised around the participating metabolites rather than the catalysing enzymes.
  • Human1 GEM reactions grouped by participating metabolite — useful when ranking metabolite-feature association results into the broader metabolic context.
  • Human1 GEM reactions grouped by metabolic subsystem (e.g., glycolysis, TCA cycle, amino-acid metabolism) — the most commonly used GEM library when ranking flux-feature association results.

The same fgsea::fora() (ORA) and fgsea::fgsea() (GSEA) implementations described above are used; the only thing that changes is which annotation object the library is loaded from. Path-collapse (redundancy removal) is also supported via the corresponding fgsea::collapsePathways() / collapsePathwaysORA() helpers, so users can collapse highly overlapping reaction sets when the resulting hit list is long.
